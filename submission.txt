├── k8s/                        - Kubernetes deployment manifests
├── docker-compose.yml          - Local orchestration
└── .env                        - Environment variables


2. DATABASE MIGRATION SCRIPTS / RESOURCE FILES
---------------------------------------------
Location: Each service contains Flyway migrations in:
backend/{service}/src/main/resources/db/migration/

Key Migration Files:
- V1__init.sql - Initial schema creation for each service
- V2__add_constraints.sql - Foreign keys and indexes
- V3__seed_data.sql - Sample data (doctors, patients, admin)

PostgreSQL Databases (per service):
- mediconnect_admin_db
- mediconnect_appointment_db
- mediconnect_auth_db
- mediconnect_doctor_db
- mediconnect_notification_db
- mediconnect_patient_db
- mediconnect_payment_db
- mediconnect_prescription_db
- mediconnect_telemedicine_db

Flyway Migration Strategy:
- Automatic migration on service startup
- Version-controlled schema evolution
- Per-database isolation
